GIT 개념 정리.
1. 개념 정리
   1) '버전 관리를 한다' 라는 말
       : 프로젝트 상태가 변경되는 정보를 알고 있다는 것이고 git은 commit을
                              사용해서 버전이 달라지는 것을 관리함
   2) '버전 관리를 잘한 commit에 포함된 정보'
      : 어떤 작업을 했는지 알려주는 작업 내역인 commit 메시지, 누가(author),
        commit 한 시간이 필요
   3) 실습의 순서
      : 기존 프로젝트를 git프로젝트로 만든다 -> 텍스트 파일들을 add(staging)을 하고
       commit 메세지를 작성해서 올리기.
